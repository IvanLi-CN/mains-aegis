# 准备与范围

在开始复刻前，先把预期摆正：当前仓库更像一套**开放的设计与实现事实库**，而不是已经收口为“完整硬件产品包”的发布物。

## 你应该具备什么前提

至少建议你具备以下能力或条件：

- 能阅读原理图/网表与器件数据手册。
- 能处理 SMD 焊接、返修与基本板级检查。
- 有基础测试条件：可调电源、万用表，最好再有示波器/逻辑分析仪。
- 愿意自己完成固件工具链安装、烧录与日志观察。
- 对锂电池硬件带来的安全边界有基本认知，不把“可拆装”理解为可以随意热插拔。

## 仓库现在已经有什么

- 系统级设计文档：BMS、充电、UPS 输出、监测、音频、开机自检等。
- 主板 / 前面板网表提炼文档。
- `ESP32-S3` 固件工程与 bring-up README。
- 大量本地化后的 datasheet / manual / reference design。

## 仓库现在还没有什么

- 一份明确冻结、可直接下单的完整 BOM 总表。
- 官方成品用户手册。
- 已经打包好的量产制造文件发布入口。
- 对所有候选器件都做完最终决策的统一结论。

因此，复刻时你必须接受这样一种工作方式：**站点手册负责指路，真正的细节事实仍在仓库原始文档里。**

## 开始前建议先锁的几个决策

### 1. 输出版本

当前项目长期围绕 `12V` / `19V` 两种主输出目标讨论与实现。你在开始做硬件与固件准备时，最好先明确自己要跟哪条版本走。

### 2. 你接受多少“候选态”

有些模块已经足够明确，适合跟进；有些模块仍需样机验证。复刻前请先确认自己是：

- 只跟进已冻结主线；
- 还是愿意一起承受候选方案的验证成本。

### 3. 复刻目标是什么

- 只是想把主板/前面板点亮、看到自检与 Dashboard？
- 还是要进一步验证充电、BMS、主输出与保护？

不同目标，会决定你需要先看哪些专题文档。

## 一条实用建议

不要一上来就追求“完整复刻全部能力”。更稳的做法是先把目标拆成可验证的小闭环：

1. 板级无短路、关键连接正确。
2. 主控能烧录、能出日志。
3. 前面板能显示、能进入自检。
4. 再逐步验证 BMS、充电、主输出与交互页。
