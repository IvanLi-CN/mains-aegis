# 稳压输出模块主动降额与主动停机（#frsr9）

## 状态

- Status: 已完成
- Created: 2026-03-16
- Last: 2026-03-16

## 背景 / 问题陈述

- 当前稳压输出模块已经具备 `BMS not ready`、`THERM_KILL_N` 与 `TPS fault` 的统一门控，但仍缺少运行态“主动降额”能力。
- 两路 `TPS55288` 在并联输出场景下可能出现单路先进入 `CC` 的情况；若长期维持高热点温度或高输出电流，当前固件只能等硬件保护或 `STATUS.OCP` 触发后再整体关断，策略过于被动。
- 需要新增一条“软保护”链路：当温度或输出电流持续超阈值时，先逐步下调 `IOUT_LIMIT`，让负载通过 `CC` 环自然掉压；若实际输出电压继续下降到保护下限，则主动关断输出并进入显式恢复路径。

## 目标 / 非目标

### Goals

- 为稳压输出模块新增“主动降额 + 主动停机”运行态状态机，与现有 `BMS/TPS/THERM_KILL` 门控并存。
- 温度与电流分别作为独立触发门限，分别计时、分别记录原因，但共享同一套 `ILIM` 递降执行链路。
- 主动降额只调整 `IOUT_LIMIT`，不主动改动 `VOUT` 设定值。
- 当降额期间输出电压低于保护下限持续一段时间后，主动关断输出并要求显式恢复。
- 保护解除后，输出仍不得自动恢复；继续沿用 `request_output_restore()` 显式恢复路径。
- 补充必要文档与最小纯逻辑验证，覆盖温度/电流触发、递降、停机、恢复前状态。

### Non-goals

- 不实现新的前面板/串口恢复入口。
- 不改变 `TPS55288` 的硬件 `THERM_KILL_N -> TPS_EN` 硬停机链路。
- 不把“正常进入 CC 模式”直接当作 `TPS fault` 语义改写；本轮只新增一条更早的主动降额链路。

## 接口变更（Interfaces）

- `output::Config`：新增主动降额/主动停机阈值与时序参数。
- `output::PowerManager`：新增内部主动保护状态机与运行时 `ILIM` 覆盖值。
- 输出门控原因扩展为包含“主动保护停机”总类；具体停机原因（温度 / 电流 / 双原因）由内部状态记录并用于日志。

## 默认参数（本轮冻结）

- 温度降额进入阈值：`40.0C`（复用 `TMP112 TLOW`）
- 温度退出阈值：`38.0C`
- 温度超阈值保持时间：`5s`
- 电流降额进入阈值：`3250mA`（默认 `3.5A` profile 下，预留约 `250mA` 裕量）
- 电流退出阈值：`3000mA`
- 电流超阈值保持时间：`3s`
- 每次 `ILIM` 递降步长：`250mA`
- 每次递降最小间隔：`2s`
- 递降最小 `ILIM`：`1000mA`
- 主动停机电压阈值：`14000mV`
- 低压持续判定时间：`2s`

> 说明：后续若默认输出 profile 不再固定为 `19V / 3.5A`，这些阈值需要重新评估并从编译期常量迁移到 profile 相关配置。

## 验收标准（Acceptance Criteria）

- Given 任一路 `TMP112` 温度连续高于 `40C` 达 `5s`，When 输出仍处于活动状态，Then 固件开始按 `250mA` 台阶递降 `IOUT_LIMIT`，且不直接改写 `VOUT` 设定值。
- Given 任一路输出电流连续高于 `3250mA` 达 `3s`，When 输出仍处于活动状态，Then 固件开始按相同步长递降 `IOUT_LIMIT`。
- Given 温度与电流同时满足降额条件，When 状态机运行，Then 日志中必须保留双原因信息，但 `ILIM` 递降链路只执行一次。
- Given 已进入主动降额，When 温度和电流都恢复到退出阈值以下，Then 运行时 `ILIM` 恢复到配置默认值。
- Given 主动降额期间任一路活动输出电压持续低于 `14V` 达 `2s`，When 状态机升级，Then 输出被主动关断，并进入“需要显式恢复”的保护状态。
- Given 主动保护停机已经解除触发条件，When 没有调用 `request_output_restore()`，Then 输出不得自动恢复。

## 具体验证场景（Validation Scenarios）

- 温度超阈值 5s 后开始降额；温度回到 38C 以下后恢复默认 `ILIM`。
- 电流超阈值 3s 后开始降额；持续高负载导致 `VOUT < 14V` 超过 2s 后进入主动停机。
- 同时温度/电流超阈值时，保护原因显示为双原因，不重复递降。
- 主动停机解除后只进入 recoverable，不自动重新使能输出。
