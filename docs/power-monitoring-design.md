# 电源监测与保护设计（UPS VIN + 双路 TPS55288 输出）

本文档描述本项目的“电压/电流遥测 + 温度采样 + 硬件级告警/保护 + 控制闭环”方案，重点覆盖：

- 三路电压/电流采样：`UPS VIN`（`7 mΩ` 分流）+ `TPS55288 OUT-A`（`10 mΩ`）+ `TPS55288 OUT-B`（`10 mΩ`）
- 快速欠压告警：以 `VIN` 下跌为触发条件（典型：下跌 `≥ 500 mV` 触发控制动作）
- 三路过流告警：分别对 3 路电流阈值进行硬件告警，并由 MCU 区分通道与执行保护/控制
- 温度采样：为功率器件与分流电阻提供温度监控输入（用于限功率/告警/保护）

> 相关电源拓扑与 TPS55288 并联注意事项见：`docs/ups-output-design.md`。

---

## 1. 信号与约束

### 1.1 三路测量对象

| 通道 | 节点 | 分流电阻 | 备注 |
|---|---|---:|---|
| CH1 | `UPS VIN` | `7 mΩ` | UPS 输入侧电流；也用于 `VIN` 欠压检测基准 |
| CH2 | `TPS55288 OUT-A` | `10 mΩ` | 两路并联输出的其中一路（独立分流） |
| CH3 | `TPS55288 OUT-B` | `10 mΩ` | 两路并联输出的其中一路（独立分流） |

### 1.2 边界条件（已知）

- 总线电压：`≤ 20 V`
- 电流范围：绝对峰值 `8 A`；实际运行：`VIN ≤ 7 A`，`OUT-A/OUT-B ≤ 6.5 A`
- 采样更新：`≤ 100 Hz`（MCU 读取/上报频率目标）
- 关键告警：`VIN` 欠压要求“报告速度”高（优先硬件告警脚/中断，而非纯轮询）

## 2. 电压/电流采样：INA3221 方案

### 2.1 选型结论

使用 `INA3221` 作为三路“分流电压 + 母线电压”监测器：

- 单芯片覆盖 3 路，适配本项目“固定 1×7mΩ + 2×10mΩ”分流配置
- 满足 `≤20V` 的总线电压测量范围与电流峰值所需的分流电压量程余量
- 提供硬件告警引脚（`PV / Critical / Warning`）可实现欠压/过流的硬件级触发

本项目已本地化数据手册：`docs/datasheets/INA3221/`。

### 2.2 量程与分辨率（用于阈值设计）

在本项目分流电阻固定的前提下，关键“电流 → 分流压降”如下：

- `7 mΩ @ 7 A → 49 mV`（`8 A` 峰值 `56 mV`）
- `10 mΩ @ 6.5 A → 65 mV`（`8 A` 峰值 `80 mV`）

INA3221 的寄存器分辨率（用于固件换算）：

- `VSHUNT_LSB = 40 µV`
- `VBUS_LSB = 8 mV`

电流换算：

```
I = VSHUNT / RSHUNT
```

> 经验提醒：低电流精度通常受输入 offset 与 `VSHUNT_LSB` 影响更明显；若需要“更准”，建议在固件里做每通道 offset/增益（含 Rsense）校准，并在噪声较大时使用更长转换时间/更高平均系数来换取稳定性。

### 2.3 连接要点（避免测量偏差）

- 分流电阻到 `INA3221 IN+/IN-` 走线务必 **Kelvin 取样（四线/并行贴近）**，避免大电流铜皮压降与开关噪声耦合进测量。
- `VBUS`（bus voltage）读数对应的是每通道 `IN-` 对地电压；因此应明确 `IN-` 落点是“分流后/负载侧”还是“分流前/源侧”，并保持三个通道的定义一致，避免固件解释混乱。

## 3. 硬件级告警/保护与控制联动

### 3.1 VIN 欠压：使用 `PV (Power Valid)` 做快速触发

`PV` 引脚用于比较“所有使能的 bus 电压通道”与 `Power-Valid Upper/Lower Limit`：

- 全部高于 upper limit → `PV` 释放为高（开漏，上拉后为高）
- 任一低于 lower limit → `PV` 拉低

建议把 `PV` 接到 MCU 中断脚，并在固件初始化阶段设置阈值，使其表达“VIN 下跌 ≥ 500 mV”这一条件。

#### 阈值设置建议（适配 VOUT ≈ VIN - 200mV）

本项目 `TPS55288` 输出在“有 VIN 时”通常比 VIN 低约 `200 mV`。若直接用 `VIN_baseline - 500mV` 作为 `PV_lower`，可能会因为 `VOUT` 的固有偏置而提前触发。

更稳妥的做法：

1. 系统进入稳定态后，采样一次三路 `VBUS`，取：
   - `Vmin = min(VIN, VOUT_A, VOUT_B)`
2. 设置：
   - `PV_lower = Vmin - 500 mV`
   - `PV_upper = PV_lower + (100~200 mV)`（提供回差，避免反复抖动）

> `PV` 阈值寄存器的 LSB 为 `8 mV`，固件写入时需要按 8mV 取整。

### 3.2 三路过流：使用 `Critical` 做快速告警 + MCU 区分通道

`Critical` 引脚用于“快速过流事件”告警（按每通道分流压降阈值比较），建议接 MCU 中断脚：

- 每通道设置各自的 `Critical-Alert Limit`（以 `VSHUNT` 为阈值）
- 触发后 `Critical` 拉低；MCU 进入 ISR 后通过读取 `Mask/Enable` 寄存器的 `CF1~CF3` 标志位判定是 CH1/2/3 哪一路超限
- 如需要“保持告警直到固件确认”，可开启 `Critical` latch（`CEN`）

> 若需要“长时间过载/接近限流”这类慢事件，可额外启用 `Warning` 告警（更偏平均值/持续事件），用于降额/告警提示，而把 `Critical` 留给硬保护路径。

### 3.3 触发后的动作（保护 vs 控制）

建议在固件里把动作分层，避免把“保护”做成“闭环抖动”：

- **硬保护（Critical）**：
  - 立即：关断/降额 TPS55288（例如拉 `EN/OE`、或通过 I2C 下调限流/降低目标电压）
  - 记录：通道号、采样值、时间戳、重复次数
  - 恢复策略：需要明确是否允许自动重试（次数/间隔），或仅人工/上位机清故障
- **控制（VIN 欠压，PV）**：
  - `PV` 下降沿触发：MCU 进入“掉电补偿”逻辑，按你的策略让 TPS 输出从“VIN-200mV 跟随”切换到“升压维持目标值”（例如回到 VIN 下跌前的目标电压）
  - `PV` 回升后：根据策略回到正常跟随模式，或保持一段时间再回切（避免来回抖动）

## 4. 温度采样

本项目温度采样的目的通常不是“高精度温度计”，而是为功率器件提供：

- 过温保护（硬切/降额）
- 温度闭环降额（限流/限功率/限输出电压）
- 温升评估数据（结构与散热验证）

### 4.1 推荐采样点

- `TPS55288` 热源附近（每颗各一个点）
- 三个分流电阻附近（`7 mΩ` + `10 mΩ` + `10 mΩ`）
- UPS OUT 口附近（连接器与理想二极管功率路径）

### 4.2 传感器形态（接口选项）

- **NTC → MCU ADC**：成本低、布点灵活；适合“趋势/保护阈值”。
- **数字温度传感器（I2C）**：读数直观、抗干扰好；适合需要更稳定的阈值判断或远距离布线。
- **复用既有温度输入**：若系统已有 BMS/充电器温度采样通道可复用（例如电池侧 NTC），需确认量程与布点是否覆盖功率热源。

> 具体器件型号与曲线（例如 NTC 阻值/B 值）应在原理图阶段锁定，并把换算表/阈值写入固件配置。

## 5. 固件采样与上报（≤100Hz）

### 5.1 推荐数据路径

- 中断路径（快）：`PV`、`Critical` → MCU ISR（只做“记录 + 状态机推进”，避免在 ISR 里做耗时 I2C 访问）
- 轮询路径（稳）：定时任务 `≤100Hz` 读取 `VBUS/VSHUNT`（以及温度），做滤波与上报

### 5.2 建议的“基线值”管理

为支持“VIN 下跌 500mV”的相对判定，建议固件维护：

- `VIN_baseline`：稳定态的 VIN 参考值（可随时间缓慢更新）
- `VOUT_baseline`：正常跟随时的两路输出参考（用于判断并联均流/异常偏差）

并在关键状态切换（例如从市电供电切到电池补偿）时，冻结/重置 baseline，避免“基线跟着掉”导致逻辑失效。
