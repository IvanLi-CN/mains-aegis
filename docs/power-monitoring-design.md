# 电源监测与保护设计（UPS VIN + 双路 TPS55288 输出）

本文档描述本项目的“电压/电流遥测 + 温度采样 + 硬件级告警/保护 + 控制闭环”方案，重点覆盖：

- 三路电压/电流采样：`UPS VIN`（`7 mΩ` 分流）+ `TPS55288 OUT-A`（`10 mΩ`）+ `TPS55288 OUT-B`（`10 mΩ`）
- 快速欠压告警：任意通道 `VBUS` 欠压触发（`12V` 固件：`11V`；`19V` 固件：`18V`），MCU 需收到信号
- 过流告警：`VIN > 7A`、`OUT-A/OUT-B > 4A`、`OUT-A+OUT-B > 6.5A` 时触发，MCU 需收到信号并区分来源
- 温度采样：为功率器件与分流电阻提供温度监控输入（用于限功率/告警/保护）

> 相关电源拓扑与 TPS55288 并联注意事项见：`docs/ups-output-design.md`。

---

## 1. 信号与约束

### 1.1 三路测量对象

| 通道 | 节点 | 分流电阻 | 备注 |
|---|---|---:|---|
| CH3 | `UPS VIN` | `7 mΩ` | `IN+3=VIN_SP`，`IN-3=VIN_SN` |
| CH2 | `TPS55288 OUT-A` | `10 mΩ` | `IN+2=ISP_TPSA`，`IN-2=ISN_TPSA` |
| CH1 | `TPS55288 OUT-B` | `10 mΩ` | `IN+1=ISP_TPSB`，`IN-1=ISN_TPSB` |

### 1.2 边界条件（已知）

- 总线电压：`≤ 20 V`
- 电流范围：绝对峰值 `8 A`；保护阈值：`VIN > 7A`、`OUT-A/OUT-B > 4A`、`OUT-A+OUT-B > 6.5A`
- 采样更新：`≤ 100 Hz`（MCU 读取/上报频率目标）
- 关键告警：欠压/过流要求“报告速度”高（优先硬件告警脚/中断，而非纯轮询）

## 2. 电压/电流采样：INA3221 方案

### 2.1 选型结论

使用 `INA3221` 作为三路“分流电压 + 母线电压”监测器：

- 单芯片覆盖 3 路，适配本项目“固定 1×7mΩ + 2×10mΩ”分流配置
- 满足 `≤20V` 的总线电压测量范围与电流峰值所需的分流电压量程余量
- 提供硬件告警引脚（`PV / Critical`）实现欠压/过流的硬件级触发

本项目已本地化数据手册：`docs/datasheets/INA3221/`。

### 2.2 量程与分辨率（用于阈值设计）

在本项目分流电阻固定的前提下，关键“电流 → 分流压降”如下：

- `7 mΩ @ 7 A → 49 mV`（`8 A` 峰值 `56 mV`）
- `10 mΩ @ 6.5 A → 65 mV`（`8 A` 峰值 `80 mV`）

INA3221 的寄存器分辨率（用于固件换算）：

- `VSHUNT_LSB = 40 µV`
- `VBUS_LSB = 8 mV`

电流换算：

```
I = VSHUNT / RSHUNT
```

> 经验提醒：低电流精度通常受输入 offset 与 `VSHUNT_LSB` 影响更明显；若需要“更准”，建议在固件里做每通道 offset/增益（含 Rsense）校准，并在噪声较大时使用更长转换时间/更高平均系数来换取稳定性。

### 2.3 连接要点（避免测量偏差）

- 分流电阻到 `INA3221 IN+/IN-` 走线务必 **Kelvin 取样（四线/并行贴近）**，避免大电流铜皮压降与开关噪声耦合进测量。
- `VBUS`（bus voltage）读数对应的是每通道 `IN-` 对地电压；因此应明确 `IN-` 落点是“分流后/负载侧”还是“分流前/源侧”，并保持三个通道的定义一致，避免固件解释混乱。
- 注意 `IN-` 输入偏置电流：若在 `IN+/IN-` 前串了较大的电阻（`kΩ` 档位误贴等），可能引入明显的 `VBUS` DC 偏差；本板网表里该串阻为 `10Ω`（见 `docs/pcbs/mainboard/netlist.enet` 的 `R103..R108`）。

### 2.4 已知问题：VBUS 读数偏高（待排查）

- 在 `TPS55288` 输出与外部仪表测量一致时，`INA3221` 的 `VBUS` 读数仍可能偏高（例如 `19V` 档偏高 `~0.5–1.0V`）。
- 跟踪：`docs/plan/0007:ina3221-vbus-offset/PLAN.md`（优先以 `U22 IN- -> U22 CHGND` 的口径对比测量）。

## 3. 硬件级告警/保护与控制联动

### 3.1 欠压：使用 `PV (Power Valid)` 做快速触发

`PV` 引脚用于比较“所有使能的 bus 电压通道”与 `Power-Valid Upper/Lower Limit`：

- 全部高于 upper limit → `PV` 释放为高（开漏，上拉后为高）
- 任一低于 lower limit → `PV` 拉低

本项目使用 `PV` 表达“任意通道欠压”并触发 MCU 告警输入（`INA3221_PV`）：

- `12V` 固件：任意通道 `VBUS < 11V` → `PV` 拉低
- `19V` 固件：任意通道 `VBUS < 18V` → `PV` 拉低

> 注意：`PV` 为**开漏、电平型**告警输出；当欠压条件持续存在时，`PV` 会持续拉低，因此 `INA3221_PV` 可能长期为低（这是预期行为）。  
> `PV` 的“释放”依赖欠压条件解除（而非单纯读寄存器清除），因此不建议把 `PV` 与“需要可靠捕获的脉冲中断”共线；本项目将 `PV` 单独接到 `INA3221_PV`。

阈值写入（`VBUS_LSB = 8mV`，寄存器按 `<< 3` 对齐）：

- `12V` 固件：`PV_upper = PV_lower = 11.000V` → `0x2AF8`
- `19V` 固件：`PV_upper = PV_lower = 18.000V` → `0x4650`

> 说明：`PV` 上电默认低；当三路 `VBUS` 全部达到 `PV_upper` 后才会释放为高，之后任意通道低于 `PV_lower` 会拉低。

### 3.2 过流：使用 `Critical`（单路 + 求和）做快速告警 + MCU 区分来源

`Critical` 引脚用于“快速过流事件”告警（按每通道分流压降阈值比较），本项目同时启用三类条件：

1. **输入过流（CH3）**：`VIN > 7A`  
   - `VSHUNT = 7A × 7mΩ = 49mV`  
   - `Critical-Alert Limit (CH3) = 49mV / 40µV × (<< 3) = 0x2648`
2. **输出单路过流（CH2/CH1）**：`OUT-A > 4A` 或 `OUT-B > 4A`  
   - `VSHUNT = 4A × 10mΩ = 40mV`  
   - `Critical-Alert Limit (CH2) = 0x1F40`，`Critical-Alert Limit (CH1) = 0x1F40`
3. **输出总过流（CH2 + CH1）**：`OUT-A + OUT-B > 6.5A`  
   - 两路分流电阻均为 `10mΩ`，求和等价为：`VSHUNT_SUM = 10mΩ × (IOUT_A + IOUT_B)`  
   - `VSHUNT_SUM = 6.5A × 10mΩ = 65mV`  
   - `Shunt-Voltage Sum Limit = 65mV / 40µV × (<< 1) = 0x0CB2`  
   - `Mask/Enable.SCC2=1`、`Mask/Enable.SCC1=1`（只对 CH2/CH1 求和；不包含 CH3）

触发与区分方式：

- 任意条件满足 → `Critical` 拉低 → `INA3221_CRITICAL` 拉低
- MCU 读取 `Mask/Enable` 寄存器区分来源：
  - `CF1/CF2/CF3`：CH1/CH2/CH3 单路过流
  - `SF`：输出总过流（求和）

### 3.3 触发后的动作（保护 vs 控制）

建议在固件里把动作分层，避免把“保护”做成“闭环抖动”：

- **硬保护（Critical）**：
  - 立即：关断/降额 TPS55288（例如拉低 `THERM_KILL_N` 触发“硬停机”（双路同时停机，见 4.3）、或通过 I2C 下调限流/降低目标电压）
  - 记录：通道号、采样值、时间戳、重复次数
  - 恢复策略：需要明确是否允许自动重试（次数/间隔），或仅人工/上位机清故障
- **控制（VIN 欠压，PV）**：
  - `PV` 下降沿触发：MCU 进入欠压处理流程（按系统策略决定降额/关断/切换供电路径）
  - `PV` 回升后：按系统策略恢复

## 4. 温度采样

本项目温度采样的目的通常不是“高精度温度计”，而是为功率器件提供：

- 过温保护（硬切/降额）
- 温度闭环降额（限流/限功率/限输出电压）
- 温升评估数据（结构与散热验证）

### 4.1 推荐采样点

- `TPS55288` 热源附近（每颗各一个点；计划使用 2× `TMP112AIDRLR` 数字温度传感器）
- 三个分流电阻附近（`7 mΩ` + `10 mΩ` + `10 mΩ`）
- UPS OUT 口附近（连接器与理想二极管功率路径）

### 4.2 传感器形态（接口选项）

- **NTC → MCU ADC**：成本低、布点灵活；适合“趋势/保护阈值”。
- **数字温度传感器（I2C）**：读数直观、抗干扰好；适合需要更稳定的阈值判断或远距离布线。
- **复用既有温度输入**：若系统已有 BMS/充电器温度采样通道可复用（例如电池侧 NTC），需确认量程与布点是否覆盖功率热源。

> 具体器件型号与曲线（例如 NTC 阻值/B 值）应在原理图阶段锁定，并把换算表/阈值写入固件配置。

### 4.3 数字温度传感器（I2C）：`TMP112A`（TPS55288 热点监测）

本项目计划在两颗 `TPS55288` 附近各放置 1 颗 `TMP112AIDRLR`，用于“板级热点温度”采样（注意这不是 `TPS55288` 结温；两者会存在温差，阈值需通过样机热测试回归）。

- 总线：`I2C1`（`3.3V`）
- 地址：通过 `ADD0` 选择（同一总线最多 4 颗）
  - `TMP112A(TPS-A)`：`ADD0=GND` → `0x48`
  - `TMP112A(TPS-B)`：`ADD0=V+` → `0x49`
- 供电去耦：`V+` 就近放置 `0.1µF`（`100nF`）到 `GND`；若 `3.3V` 噪声较大/走线较长，可再并 `1µF`（或通过磁珠/小电阻做隔离再在传感器侧加电容）
- 过温硬保护（推荐）：利用 `ALERT`（开漏）做“任一路过温 → 双路 `TPS55288` 同时停机”的硬件链路，避免单路停机后负载被另一颗接手导致快速过温。
  - 模式：建议使用 `Comparator mode (TM=0)` + `POL=0`，这样 `ALERT` 在温度 ≥ `T(HIGH)` 时会持续拉低，并保持到温度 < `T(LOW)` 才释放（通过 `T(HIGH)/T(LOW)` 实现滞回；可配合 Fault Queue 做去抖）。
  - 过温汇总 + MCU 强制（同一根 GPIO，低有效）：
    - 汇总线：`THERM_KILL_N`（开漏线与；低=过温/停机请求）
    - `TMP112A(TPS-A).ALERT` —(串联 `~1kΩ`)→ `THERM_KILL_N`
    - `TMP112A(TPS-B).ALERT` —(串联 `~1kΩ`)→ `THERM_KILL_N`
    - `THERM_KILL_N` 上拉 `~10kΩ` 到 `3.3V`
    - `MCU.GPIO_THERM_KILL_N` 直接连接 `THERM_KILL_N`：
      - 作为输入：接收过温/停机请求（电平型）
      - 作为开漏输出：MCU 可拉低 `THERM_KILL_N` 强制双路停机（不走 I2C）
  - 关断执行（两路同时停机）：
    - 将 `TPS55288A.EN/UVLO` 与 `TPS55288B.EN/UVLO` 视为同一个节点 `TPS_EN`（建议各自串联 `~100Ω` 再汇聚，便于隔离与调试）
    - 使用 `BSS138PS,115`（双 NMOS）实现“反相 + 下拉”：
      - Q1（反相）：`G=THERM_KILL_N`，`S=GND`，`D=THERM_KILL_H`，并用 `~100kΩ` 上拉到 `3.3V`
      - Q2（下拉）：`G=THERM_KILL_H`，`S=GND/CHGND`，`D=TPS_EN`
    - `TPS_EN` 的上拉/分压网络按 `TPS55288.EN/UVLO` 的 UVLO 需求设计；本项目约定 **UVLO 分压参考地选用 `AGND_TPSA`**。若关断下拉（Q2）参考的是 `GND/CHGND`，则需要保证 `AGND_TPSA` 与 `CHGND` 按 `TPS55288` 的布局指南做单点/低阻连接，避免地弹噪声影响 `TPS_EN` 的阈值稳定性
  - 备注：
    - 不把 `TMP112A.ALERT` 并入 `INA3221_*` 告警线：Comparator 模式下可能长期拉低，会掩盖其它中断/告警；本方案用 `THERM_KILL_N` 独立接入 MCU，兼顾“硬停机”与“MCU 可见”。

> 资料：TMP112 datasheet（地址与 `ALERT`/模式相关章节）https://www.ti.com/lit/ds/symlink/tmp112.pdf

## 5. 固件采样与上报（≤100Hz）

### 5.1 推荐数据路径

- 中断路径（快）：`PV`、`Critical` → MCU ISR（只做“记录 + 状态机推进”，避免在 ISR 里做耗时 I2C 访问）
- 轮询路径（稳）：定时任务 `≤100Hz` 读取 `VBUS/VSHUNT`（以及温度），做滤波与上报

## 6. 固件告警阈值（本项目）

- 欠压阈值（任意通道 `VBUS` 低于阈值触发）：
  - `12V` 固件：`11V`
  - `19V` 固件：`18V`
- 过流阈值：
  - `I_OC_VIN = 7A`（CH3，`7mΩ`）
  - `I_OC_OUT_A = 4A`（CH2，`10mΩ`）
  - `I_OC_OUT_B = 4A`（CH1，`10mΩ`）
  - `I_OC_OUT_SUM = 6.5A`（CH2+CH1 求和，`10mΩ`）
