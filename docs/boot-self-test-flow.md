# 开机自检流程（Boot Self-Test）

本文档定义固件上电后的自检顺序与门控策略，目标是：

- 在不触发紧急条件时，不干扰 `TPS55288` 现有输出状态。
- 先完成基础链路与独立传感器准备，再按模块顺序探测。
- 仅禁用缺失/异常模块，避免“全局连坐”。

## 自检总原则

- **默认只读探测**：自检阶段优先做 presence/status 读取。
- **非紧急不改输出**：未命中紧急条件时，不在自检阶段主动改 `TPS55288` 输出（不主动关断/重配）。
- **紧急可关断**：仅在明确紧急条件命中时，允许在自检阶段执行 `TPS55288 disable_output()`。

## 固定执行顺序

1. `TPS55288` SYNC 启动（SYNCA/SYNCB）并得到 `sync_ok`。
2. 独立传感器配置：
   - `INA3221`：reset + config。
   - `TMP112A(0x48/0x49)`：阈值/告警配置。
3. 模块探测（固定顺序）：
   - 屏幕模块（目前以前面板 IO 扩展 `TCA6408A@0x21` 的可达性代表）
   - `BQ40Z50`
   - `BQ25792`
   - `TPS55288`
4. 根据探测结果门控模块：仅禁用缺失/异常模块。

## 模块门控策略

- **屏幕模块**：
  - 若缺失，仅禁用屏幕/前面板相关模块；不影响 BMS/Charger/TPS 处理链路。
- **BQ40Z50**：
  - 若缺失，禁用 BMS 轮询模块（`bms_addr=None`）。
- **BQ25792**：
  - 若缺失，禁用 charger 模块（维持 `CE` 安全态）。
- **TPS55288（按通道）**：
  - 基于 `desired_outputs`、`sync_ok`、独立传感器状态、通道 presence/status 联合判定。
  - 只禁用异常/缺失通道，不连坐其它通道。

## 紧急停止条件（TPS 自检阶段允许改输出的唯一入口）

命中任一条件即进入 emergency-stop：

- `THERM_KILL_N` 被拉低。
- `TPS55288 STATUS` 命中 `SCP/OCP/OVP` 任一保护位。

进入 emergency-stop 后：

- 允许执行 `disable_output()` 关断受影响（或全部）TPS 通道。
- 输出通道门控结果强制降级为 `None` 或对应安全子集。

## 当前实现备注

- 当前仓库中“屏幕模块”尚无独立运行时驱动，先以 `TCA6408A@0x21` 可达性作为屏幕链路可用性信号。
- `FUSB302B@0x22` 同属前面板链路探测项，但不替代屏幕 presence 判据。
- 如后续引入独立屏幕驱动（例如 SPI 屏控制器），应将“屏幕模块探测”替换为该驱动的真实健康检查。
