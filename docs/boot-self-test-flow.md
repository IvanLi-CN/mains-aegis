# 开机自检流程（Boot Self-Test）

本文档定义固件上电后的自检顺序、模块责任边界与初始化决策，目标是：

- 在不触发紧急条件时，不干扰 `TPS55288` 现有输出状态。
- 先完成基础链路与独立传感器准备，再按模块顺序探测。
- 在“需要输出”的模式下显式决定是否允许发起放电授权尝试，而不是把 `BMS` 阻断简单当成失败或忽略不管。
- 让自检界面上的每个条目只对自己的模块健康负责。

## 自检总原则

- **默认只读探测**：自检阶段优先做 presence/status 读取。
- **非紧急不改输出**：未命中紧急条件时，不在自检阶段主动改 `TPS55288` 输出（不主动关断/重配）。
- **紧急可关断**：仅在明确紧急条件命中时，允许在自检阶段执行 `TPS55288 disable_output()`。
- **显式决策后再动执行链路**：自检探测结束后，先做“是否允许发起放电授权尝试”的策略决策，再进入初始化应用阶段。
- **初始化后执行门控**：初始化应用阶段按模块职责与门控结果使能/禁用模块。
- **门控可转入待恢复**：运行时门控解除后，只允许把输出状态转入“可恢复未恢复”；本轮固件不自动重新使能输出。

## 固定执行顺序

1. `TPS55288` SYNC 启动（SYNCA/SYNCB）并得到 `sync_ok`。
2. 独立传感器配置：
   - `INA3221`：reset + config。
   - `TMP112A(0x48/0x49)`：阈值/告警配置。
3. 模块探测（固定顺序）：
   - 屏幕模块（目前以前面板 IO 扩展 `TCA6408A@0x21` 的可达性代表）
   - `BQ40Z50`
   - `BQ25792`
   - `TPS55288`
4. 放电授权决策：当目标模式请求输出时，明确判断是否允许发起 `BMS` 放电路径恢复尝试。
5. 初始化应用阶段：根据探测结果与授权决策门控模块。

## 条目责任边界

- **BMS 条目**只反映 `BQ40Z50` 自己的状态：
  - `OK`：通信正常，放电路径就绪
  - `LIMITED`：通信正常，但 `DSG` 路径未就绪、等待授权恢复或被策略拒绝
  - `ERR`：链路断、普通访问失败或进入 ROM/异常不可用态
- **Charger 条目**只反映 `BQ25792` 自己的状态：
  - `RUN`：当前允许充电且充电路径工作中
  - `IDLE`：芯片正常，但当前不需要或不允许充电
  - `ERR/WARN`：芯片本身链路或运行异常
- **Output 条目**只反映输出模块在“当前上游条件下”的状态：
  - `RUN`：上游已授权且 `TPS` 通道已建立
  - `HOLD`：上游 `BMS` 尚未放行，当前不评价为输出模块故障
  - `ERR/WARN`：只有在上游已放行后，`TPS` 自身失联/保护位命中才算输出异常

## 模块门控策略

- **屏幕模块**：
  - 若缺失，仅禁用屏幕/前面板相关模块；不影响 BMS/Charger/TPS 处理链路。
- **BQ40Z50**：
  - 运行地址默认固定为 `7-bit 0x0B`（等价 `8-bit W=0x16 / R=0x17`）；仅显式诊断构建（`bms-dual-probe-diag`）会额外探测 `0x16(7-bit)`。
  - 若缺失，禁用 BMS 轮询模块（`bms_addr=None`）。
  - 且作为硬门控条件：初始化后必须禁用 `TPS55288` 输出。
  - 若可通信但 `OperationStatus` 显示放电未就绪（`XDSG=1` 或 `DSG=0`），视为 `BMS LIMITED`，同样门控 `TPS55288` 输出。
  - 只要 `BMS` 普通通信正常，就不走“离线激活”语义；这类情况在启动期进入“放电授权决策”而不是“激活缺失设备”。
  - 当以下条件同时满足时，初始化阶段允许自动发起一次放电授权恢复尝试：
    - 当前模式确实请求输出
    - `BQ40Z50` 已在线
    - `no_battery != true`
    - `RCA alarm != true`
    - `discharge_ready == false`
    - `THERM_KILL_N` 未断言
    - `BQ25792` 与输入电源路径可用
    - `Type-C / charger` 已确认输入存在；这一步不要求 `INA3221 CH3` 的 `VIN` 采样已经先稳定
  - 授权恢复尝试使用现有的 `BQ25792` 辅助恢复链路；其目的是恢复放电路径，不代表“离线 BMS 激活”。
  - 这条恢复链路只有在 `discharge_ready == true` 时才算成功；若只是普通寄存器读通、但放电路径仍未释放，则流程继续保持 `LIMITED/HOLD`，直到阶段耗尽或超时收口。
  - 运行时若 BMS 恢复到放电就绪（`XDSG=0` 且 `DSG=1`），仅清除 `bms_not_ready` 门控；若 `VIN` 在线则进入“可恢复未恢复”，仍需后续显式 restore 请求才重新使能输出。
  - `BQ25792` 充电默认同样禁用；仅当固件以 `force-min-charge` 特性构建时，允许保留 charger 模块并以最小电流做唤醒尝试（不改 `VREG`）。
- **BQ25792**：
  - 若缺失，禁用 charger 模块（维持 `CE` 安全态）。
  - 若芯片正常但电池路径当前未允许充电，不应把 charger 条目标成故障；显示应为 `IDLE`。
- **TPS55288（按通道）**：
  - 基于 `desired_outputs`、`sync_ok`、独立传感器状态、通道 presence/status 联合判定。
  - 只禁用异常/缺失通道，不连坐其它通道。
  - 当 `BQ40Z50` 缺失时，不论通道自身探测结果如何，初始化后统一禁用。
  - 当 `BQ40Z50` 在线但放电路径未授权时，`TPS` 条目显示 `HOLD`；只有放电已授权后，`TPS` 的 `ERR/WARN` 才表示输出模块自身异常。

## 紧急停止条件（TPS 自检阶段允许改输出的唯一入口）

命中任一条件即进入 emergency-stop：

- `THERM_KILL_N` 被拉低。
- `TPS55288 STATUS` 命中 `SCP/OCP/OVP` 任一保护位。

进入 emergency-stop 后：

- 允许执行 `disable_output()` 关断受影响（或全部）TPS 通道。
- 输出通道门控结果强制降级为 `None` 或对应安全子集。

## 当前实现备注

- 当前仓库中“屏幕模块”尚无独立运行时驱动，先以 `TCA6408A@0x21` 可达性作为屏幕链路可用性信号。
- `FUSB302B@0x22` 同属前面板链路探测项，但不替代屏幕 presence 判据。
- 自检页只有在“本模式必需模块均已 clear”时才切入 dashboard；若 `BMS LIMITED` 或 `Output HOLD`，页面会继续停留在自检页。
- 如后续引入独立屏幕驱动（例如 SPI 屏控制器），应将“屏幕模块探测”替换为该驱动的真实健康检查。
