# 充电器设计（BQ25792 + Type‑C/PD 控制器（需 PPS），双输入）

本文档描述本项目基于 **TI BQ25792**（buck-boost 充电管理）实现的充电器方案：主 USB‑C + DC5025（直流输入）双输入；支持 1A / 500mA / 100mA 三档充电电流；支持 BQ25792 OTG（VBUS 输出）能力；不使用 Ship MOSFET；充电器电路 + BMS + `4S1P` 电池组共板（`10 cm × 10 cm` 四层 PCB），电芯通过 `4×21700` 电池座连接，可免工具拆装。

**重要约束（项目硬性需求）**：主 USB‑C 口需要同时支持 **PPS 输入（Sink PPS/APDO）** 与 **PPS 输出（Source PPS/APDO）**，因此 Type‑C/PD 控制器必须明确支持 PD3.0 PPS（或具备可验证的 PPS 实现路径）。

> 电池包与 BMS 设计背景：`docs/hardware-selection.md`、`docs/bms-design.md`  
> BQ25792 资料：`docs/datasheets/BQ25792/BQ25792.md`  
> CH224Q（备选，仅受电）资料：`docs/datasheets/ch224q/CH224Q.md`
> FUSB302B（参考：数据手册标称 PD2.0，不满足本项目 PPS 需求）资料：`docs/datasheets/FUSB302B/FUSB302B.md`

## 1. 需求与边界

### 1.1 已知需求

- 电池包：`4S1P`，满充 `16.8V`，截止放电 `10.0V`（见 `docs/hardware-selection.md`）
- 充电电流档位：
  - 常规：`1.0A`（默认）
  - 预留：`2.0A`（硬件能力预留，后续可通过固件放开）
  - 低速：`500mA`
  - 超低速：`100mA`
- 双输入：
  - `VIN1`：主 USB‑C 口（Type‑C/PD 控制器负责 CC/PD 协商：固定 PDO + PPS/APDO；可选把 `D+/D-` 通过 `CH442E` 切到 `BQ25792 D+/D-` 做 DPDM/传统适配器检测）
  - `VIN2`：DC5025 圆孔 DC 输入（纯输入）
- OTG：使用 `BQ25792` 的 OTG（VBUS 输出）能力；主 USB‑C 口的 Type‑C/PD 角色由 Type‑C/PD 控制器 + MCU 管理
- 不使用 Ship MOSFET
- 结构/PCB 约束：充电器电路 + BMS + `4×21700` 电池座共板（`10 cm × 10 cm` 四层 PCB），电芯可免工具拆装（需考虑“无电池/电池被断开”场景）

### 1.2 关键结论（可行性）

在不引入额外大改的前提下，**“双输入 + PD 20V 受电 + 三档充电电流 + OTG 输出”整体可做**，但需要明确两个关键边界：

1. **BQ25792 是 NVDC power-path 充电器**：其 `SYS` 节点是 buck-boost 转换器的系统输出；设计中应避免把项目的“高电流主放电路径/大功率 DCDC”接到 `SYS` 上，否则会受 BQ25792 内部功率路径能力限制（这与项目约 `120W` 级输出电流量级不匹配）。  
   - 推荐做法：把 BQ25792 作为“充电/USB 口电源管理”子系统，`SYS` 只做本子系统的局部电容与测量节点；项目的大功率输出仍从电池包主回路获取。
2. **USB-C 口“OTG”是否要在同一个 Type‑C 口上实现**：  
   - 若只需要 **VBUS 供电输出（例如 USB‑A 口或专用 5V 输出口）**：BQ25792 OTG 可直接实现（推荐方案）。  
   - 若要求 **同一个 USB‑C 口同时承担受电 PD（Sink）+ OTG 供电（Source）**：需要 Type‑C/PD 通信能力与角色切换控制，且本项目要求 **PPS 输入 + PPS 输出**。供电路径仍由 `BQ25792` 负责。

### 1.3 待你确认（否则只能按假设落图）

1. **DC5025 输入**：已确认为 `12V` 或 `19V`，最大不超过 `20V`（因此全套耐压按 `25V` 档位设计即可）；且明确会发生“热插拔 > 15V”场景，因此 VIN2 需要按 BQ25792 典型应用图（Figure 10‑1/10‑2，Note 1）的建议加入输入 RC 阻尼网络（见 5.2）。  
2. **OTG 的物理输出端口**：  
   - A. 独立 USB‑A（5V）/专用输出口（推荐）  
   - B. 与受电同一个 USB‑C 口（本项目：Type‑C/PD 控制器 + `ESP32‑S3` 管理 Type‑C/PD 角色与协商，含 PPS）  
3. **充电时是否允许系统大功率输出同时工作**：若允许，需要做系统级功率预算与热设计；否则可在固件策略中“插电充电时禁用大功率输出”。

## 2. 系统方案

### 2.1 方框图（逻辑）

```
USB-C Receptacle ── CC1/CC2 ── Type-C/PD Controller (DRP + PD3.0 PPS, policy by ESP32-S3 or built-in)
         │
         ├─ D+/D- ── CH442E ───┬─ ESP32-S3 USB (GPIO19/20, USB2 data)
         │                    └─ BQ25792 D+/D- (DPDM / legacy detect, optional)
         │
         └─ VBUS ── Input FETs (ACFET1/RBFET1) ──┐
                                                  ├─ BQ25792 VBUS/PMID ── buck-boost ── SYS (local)
DC5025 Jack ───────── Input FETs (ACFET2/RBFET2) ┘                           │
                                                                             └─ BATFET ── BAT ── Battery Pack (4S, via BMS CHG path)

MCU (host) ─ I2C ─ BQ25792              (set charge/OTG/input select)
MCU (host) ─ I2C ─ Type-C/PD Controller (attach/orientation/role/PD+PPS)
```

### 2.1.1 与电池包/BMS 的连接边界（必须明确）

- `BQ25792 BAT / BATP`：连接到 **电池包正端（PACK+）**（也就是 BMS 充电通路允许电流流入的节点）。  
  - 若你希望 BMS 在故障时能够硬件切断充电电流，**不要**把 BQ25792 直接接到电芯正端（CELL+）去绕过 CHG FET。
- `BQ25792 GND`：连接到电池包地（PACK- / 系统地），并遵循 TI 的功率回流/开关回路布局要求。
- `PACK+ / PACK-` 端口防护：在电池座/端子处并联 `SMBJ18CA`（双向 TVS，`DO-214AA`），用于 ESD/瞬态钳位（详见 `docs/bms-design.md` 的 PACK 端 ESD/TVS 说明）。
- 约 `120W` 级的主功率输出（例如电池到 19V 的大功率 DCDC/项目文档中的“OTG 19V”）应从电池主回路获取，**不要**走 BQ25792 的 `SYS` 节点。

### 2.2 输入选择策略

- `VAC1` 定义为 USB-C 输入（默认优先），`VAC2` 定义为 DC5025 输入。  
  BQ25792 支持双输入选择；推荐同时上 `ACFET/RBFET`，避免任一输入反灌另一输入。
- 固件策略建议：
  - 若两路同时存在，优先级按项目需求确定（默认 USB 优先 or DC 优先均可，BQ25792 默认 VAC1 为主输入）。
  - 切换输入源前先将 `EN_HIZ` 置位/拉低 `ILIM_HIZ` 进入非开关模式，完成后再恢复，降低毛刺与反向电流风险。

### 2.3 充电档位策略

#### 2.3.1 充电电压

- `4S` 目标：`VREG = 16.8V`（与电芯 `4.2V/cell` 对齐）。BQ25792 的 `PROG` 选 `4S`，默认 `VREG=16.8V`。

#### 2.3.2 充电电流

BQ25792 充电电流寄存器分辨率为 `10mA`（范围 `50mA~5000mA`），因此 `1A/500mA/100mA` 可直接落到寄存器配置。

推荐把“低速/超低速”作为固件可切换策略，而不是硬件固定：

- 预留：`ICHG=2000mA`（硬件能力预留；默认固件不放开）
- 常规：`ICHG=1000mA`
- 低速：`ICHG=500mA`
- 超低速：`ICHG=100mA`

输入功率约束要点（决定“2A 是否跑得满”）：

- BQ25792 的输入限流 `IINDPM` 上限约 `3.3A`（见 `docs/datasheets/BQ25792/BQ25792.md` 特性描述），因此在 `4S` 满充 `16.8V` 时：
  - `20V` 输入：`2A@16.8V` 约 `33.6W`，输入电流约 `~1.7A`（加损耗略升），一般可跑满。
  - `12V` 输入：输入电流约 `~3A`（加损耗接近 `3.3A` 上限），多数情况下可跑满但裕量不大；若触发 `IINDPM/VINDPM`，BQ25792 会自动降充电电流。
  - `5V` 应急输入：想跑 `2A@16.8V` 需要 `~7A` 级输入电流，不现实；应急模式下直接把 `IINDPM/ICHG` 降到低档位（例如 `500mA/100mA`，或按实际 5V 适配器能力设定）。

> 注：当输入功率受限（例如非 PD 的 `5V` 口、或 BC1.2 检测到 `SDP 500mA`），BQ25792 会进入 DPM（动态电源管理）自动限流；固件也应基于 `VBUS`/`IBUS` ADC 与 `IINDPM/VINDPM` 状态主动降档，避免反复打嗝。

## 3. 主 USB‑C：Type‑C / PD（需 PPS）设计

### 3.1 能力边界（必须明确）

- 主 USB‑C 口必须支持：  
  - **PPS 输入（Sink APDO/PPS）**：与适配器协商 APDO 合同，并按策略请求电压/电流  
  - **PPS 输出（Source APDO/PPS）**：对外发布 APDO，并按对端请求调节输出  
- BQ25792 的职责：提供电源转换与 power-path（受电/充电/OTG）；在 OTG 时可输出 `2.8V~22V` 且 `10mV` 步进，满足 PPS 输出侧电压可调的执行条件（前提是 Type‑C/PD 控制器与固件能正确建立/管理 PPS 合同）。
- Type‑C/PD 控制器的职责：CC/方向/角色切换 + PD 合同（含 PPS/APDO）建立与维持；并与 MCU/BQ25792 协调输出电压/限流与互锁时序。

> 备注：本仓库已入库 `FUSB302B` 数据手册，但其文档标称 **PD2.0**（见 `docs/datasheets/FUSB302B/FUSB302B.md`），无法作为“PPS 输入 + PPS 输出”的主方案依据，因此仅作为参考资料保留。

### 3.2 关键连接（主 USB‑C 口）

- `CC1/CC2`：USB‑C 口 CC → Type‑C/PD 控制器（含插入检测、方向识别、Type‑C 电流声明/侦测、PD 消息通道）
- `SDA/SCL/INT_N`：`ESP32‑S3` ↔ Type‑C/PD 控制器（若为 I2C/中断型）
- `VBUS`：作为 Type‑C/PD 控制器的 `VBUS` 电压感知（以及 `BQ25792` 的输入电源）
- `D+/D-`：若主 USB‑C 口需要 **USB2.0 数据通信**，则 `D+/D-` 默认连接到 `ESP32‑S3`（`GPIO19/20`）。  
  若仍需兼容“非 PD / 传统适配器”的 DPDM 检测，可通过 `CH442E` 把 `D+/D-` 临时切换给 `BQ25792 D+/D-`（DPDM）。

### 3.3 备选：CH224Q（仅受电，非 DRP）

若你希望在“固件还没跑起来/不想做 PD 协议栈”的情况下，依然能从部分 PD 适配器获取固定高压输入，可选用 `CH224Q` 作为 **Sink‑only** 受电触发方案。

注意：

- `CH224Q` 与主 USB‑C 的 Type‑C/PD 控制器 **不能同时并联到同一个 Type‑C 口的 CC1/CC2**；若要上 `CH224Q`，需用 `0Ω` 跳线在两者之间二选一（默认选主口控制器）。
- 该方案不覆盖同口 OTG/DRP 需求；对“同口充电 + OTG + USB2 数据”不适合作为主方案。
- `CH224Q` 的电压/配置细节以 `docs/datasheets/ch224q/CH224Q.md` 为准；本项目 `BQ25792` 输入侧按最高 `20V` 设计，不做更高电压请求。

## 4. BQ25792 设计

### 4.1 PROG 与默认配置

为确保在 MCU 未介入时也不会把 `4S` 当成其他串数，建议 `PROG` 采用 `4S` 配置：

- `PROG = 17.4kΩ` → `4S` + `1.5MHz`（默认 `ICHG=1A`、`VSYSMIN=12V`、`VREG=16.8V`）

### 4.2 “无电池/电池被断开”安全策略（强制）

电芯可拆卸且 BMS 可能在故障时断开电池，因此必须保证：

- `CE` 引脚 **默认拉高**（充电禁用），由 MCU 在确认电池存在且 BMS 允许充电后再拉低使能。
- 在检测到电池被断开或 BMS 关闭充电通路时，固件应立即：`EN_CHG=0` 或将 `CE` 拉高，避免 BAT 浮空导致 SYS 过冲反复触发保护。

### 4.3 双输入外部 MOSFET（ACFET/RBFET）

推荐实现方式：两路输入各用一对背靠背 N‑MOS（common‑source），由 `ACDRV1/2` 驱动，达到：

- 输入选择
- 反灌/反接阻断（尤其是 DC 口）
- 热插拔时更可控（可配合 TVS/浪涌设计）

连接要点（按 BQ25792 引脚定义约束）：

- `VAC1` 接 USB‑C 输入侧（ACFET1/RBFET1 之前）用于“输入存在”检测；`VAC2` 同理接 DC 输入侧。
- `VBUS` 接两路输入 MOSFET 选通后的汇合点（即真正给 BQ25792 供电的输入端）。

### 4.4 OTG（VBUS 输出）

BQ25792 支持在无输入时，从电池向 `VBUS` 输出可调 `2.8V~22V`，并提供 OTG 输出电流调节能力。

**推荐落地方式**：把 OTG 输出接到独立 USB‑A/专用输出口（5V），避免与同一 USB‑C 口的受电 PD 逻辑冲突（详见 8.2）。

| 器件 | 建议值/范围 | 选型要点 |
|---|---|---|
| 电感 L | `1.0µH`（1.5MHz）或 `2.2µH`（750kHz） | 依据 BQ25792：`1.5MHz ↔ 1µH`，`750kHz ↔ 2.2µH`；`Isat` 建议 ≥ 6A（按 `2A` 预留 + `IINDPM` 上限留余量）。示例（来自 BQ25792 典型曲线条件）：`SPM6530T-1R0M120`（1µH）/ `74437346022`（2.2µH） |

| SYS 电容 | `0.1µF + ≥5×10µF`（X7R/X5R，耐压 ≥ 25V） | 4S 场景 SYS 可到 `18.8V`（VREG range 上限），耐压需上到 25V；同样注意降额 |
| BAT 电容 | `2×10µF`（X7R/X5R，耐压 ≥ 25V） | 贴近 BAT 引脚；配合 BATP 100Ω |
| BTST1/2 | `47nF`（≥10V） | bootstrap 电容，贴近引脚 |
| REGN 电容 | `4.7µF`（≥10V） | REGN LDO 去耦，贴近引脚 |
| BATP 串阻 | `100Ω` | BQ25792 建议在 BATP 与电池正之间串 100Ω |

#### 5.3.1 关于“10µF 换 22µF/100µF”的建议（关键点）

- **输入/输出去耦优先用 MLCC（X7R/X5R）**：BQ25792 数据手册对 `VBUS/PMID` 与 `SYS` 都是“低 ESR 陶瓷电容优先”，并给出了 `0.1µF + N×10µF` 的参考组合（本表即按其建议汇总）。
- **可以换，但要按“有效电容”算**：高压下 MLCC 直流偏压降额很明显，`10µF/25V` 在 `20V` 附近的有效容量往往会大幅下降；用 `22µF`（更大封装/更高耐压）来“换回有效值”是常见做法。是否够用以你选型的电容曲线为准（建议优先选能提供 DC‑bias 曲线的系列）。
- **不建议“单颗很大”替代**：`100µF` MLCC 可用，但更建议多颗并联（例如多颗 `10~22µF`）来降低 ESL/分摊应力，并降低单颗 MLCC 机械开裂风险。
- **USB‑C 口边上的 VBUS 电容别无脑加大**：Type‑C/PD 对受电端热插拔浪涌有要求；建议把“堆电容”的重心放在 **输入 MOSFET 后的 PMID/VBUS 汇合点**（更可控），接口处以抑制尖峰/EMI 为主（小电容 + TVS）。

### 5.4 输入通路 MOSFET（ACFET/RBFET）

建议按 BQ25792 推荐的“每路一对背靠背 N‑MOS”实现：

- `VDS`：≥ 30V（建议 40V 以覆盖热插拔尖峰裕量）
- `RDS(on)`：尽量低（在 `VGS≈5V` 条件下评估）
- 热：按 `IINDPM` 上限（最多 3.3A）和布局铜皮评估温升
- 封装建议：`DFN3x3 / PowerPAK / SO-8FL` 之类带散热焊盘的封装优先

> 备注：如果你确认 DC 输入也只到 12~20V 且电流上限较低，可用更小封装/更高 RDS(on) 以换取成本。

### 5.5 温度检测（TS）

BQ25792 的 `TS` 需要 NTC 分压；项目的 BMS 也有温度采样。推荐：

- 使用同一颗电池包 NTC（典型 10k NTC，曲线需与你的温度策略一致）
- 分压网络按 BQ25792 推荐方案设计，温度窗由硬件分压 + 寄存器配置共同决定

### 5.6 D+/D- 与 CC1/CC2 的“功能归属”建议（避免冲突）

- `D+/D-`（主 USB‑C 口 USB2 数据/DPDM）：  
  - **不要**把 `ESP32‑S3 USB D+/D-` 与 `BQ25792 D+/D-（DPDM）` 硬并联在同一对线上。  
  - 推荐用 `CH442E` 做二选一切换：默认连 `ESP32‑S3`；仅在需要 DPDM 识别时临时切到 `BQ25792`。
- `CC1/CC2`（Type‑C/PD）：默认只连接到主 USB‑C 的 Type‑C/PD 控制器（并做 ESD），走线对称/等长/远离开关节点。  
  若改用 `CH224Q` 备选方案，则需通过 `0Ω` 跳线在两者之间二选一，避免 CC 并联冲突。

## 6. 关键布局要点（强烈建议照做）

1. `PMID/SYS` 的 `0.1µF` 必须贴近芯片，且回地环路最小（BQ25792 数据手册强调其噪声滤波关键性）
2. `SW1/SW2` 到电感走线短且粗，尽量减小辐射面积；避免在其上方/下方穿敏感信号
3. `BATP` 走线远离 `SW` 节点；`BATP` 串 `100Ω` 贴近芯片
4. `D+/D-` 与 `CC1/CC2` 贴近接口做 ESD，走线避免贴近开关节点
5. 输入 TVS 与输入电容靠近接口放置，抑制热插拔尖峰

## 7. 固件/寄存器建议（用于落地三档电流）

### 7.1 BQ25792（概要）

最小可行固件流程：

1. 上电保持 `CE=HIGH`（禁充）
2. 通过 ADC/寄存器确认电池存在、无 TS fault、BMS 允许充电
3. 选择输入源（VAC1/VAC2），设置 `IINDPM/VINDPM`
4. 根据档位设置 `ICHG=2000/1000/500/100mA`（默认不启用 2A，作为预留）
5. 拉低 `CE` 并置位 `EN_CHG`

### 7.2 Type‑C/PD（主 USB‑C）

- 固件实现：Type‑C attach/detach、方向识别、DRP/角色切换；以及 PD 合同建立（固定 PDO + PPS/APDO）。
- “无 MCU”兜底：仅保证默认 `5V` 受电（Dead Battery/Type‑C 默认供电）；不应在 MCU 未运行时期望高压 PD 合同成立。

## 8. 风险与可升级路径

### 8.1 风险清单（需要提前决策）

1. **同口 DRP/PD 的系统复杂度**：主 USB‑C 同时支持受电+OTG 时，必须保证“角色切换、VBUS 供电方向、过流/反灌保护、以及 USB2 数据通路切换”的时序与互锁正确。
2. **DC 输入热插拔尖峰**：虽然 DC 额定 `12/19V` 且最大不超过 `20V`，但线缆热插拔仍可能产生尖峰；需通过 TVS + 接口处电容 + 输入 FET 位置/走线来抑制，避免误触发保护或 EMI 问题。
3. **插电同时大功率输出**：需要系统级功率预算与散热，否则会出现“充电被 DPM 拉垮/反复启停/温升过高”。

### 8.2 若必须同一个 USB‑C 口实现受电+OTG

本项目目标方案为“Type‑C/PD 控制器（需 PPS）+ 主控协议/策略 + BQ25792 供电路径”的架构：

- Type‑C/PD 控制器：负责 CC/方向识别、VCONN、PD 收发与 PPS/APDO 合同管理（取决于器件能力/架构）
- `ESP32‑S3`：负责 PD 协议/策略（固定 PDO + PPS；以及角色切换）；并与 `BQ25792` 协调 OTG 输出/输入限流/保护策略
- `CH224Q`：仅适合“低成本固定受电触发（Sink-only）”备选；不适合作为同口 DRP 的核心控制器

由于本项目硬性要求 **PPS 输入 + PPS 输出**，Type‑C/PD 控制器的选型必须以“明确支持 PD3.0 PPS（APDO）”为前提条件。

## 9. 验证清单（bring‑up）

- 双输入互斥：USB 与 DC 同时接入时无反灌、无异常发热
- PD 受电：插入 PD 适配器可稳定协商到 `12/15/20V`（按固件策略）；插入非 PD 源时能稳定得到默认 `5V`
- 充电三档：`1A/500mA/100mA` 可稳定进入 CC/CV；热调节/安全计时器行为可复现
- 预留 2A：在 `12~20V` 输入下可进入 `2A` 档（若受 `IINDPM/VINDPM` 约束会自动降流，行为符合预期）
- “无电池”场景：电池拔掉后不会反复触发保护；固件能检测并保持禁充
- OTG：无输入时 OTG 输出稳定，过流/短路保护有效
